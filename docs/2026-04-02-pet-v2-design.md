# 017Pet V2 设计文档

> 日期：2026-04-02
> 状态：设计讨论中
> 参与者：谷雨、Claude、Codex

## 背景与目标

017Pet 当前是一个功能完整的微信电子宠物 Agent（5属性、探险、成就、日记、收藏、周报），但所有用户领养的都是同一只统一性格的企鹅。V2 的目标是把它从"一个有趣的 Agent demo"升级为"一个值得传播的产品"。

### 四个核心目标（按优先级）

1. **社交货币**：用户愿意截图发朋友圈，别人看了想参与
2. **养成成就感**：每只宠物独一无二，是"我养出来的"
3. **陪伴感**：每天都有一点小变化，它一直在那里
4. **情绪出口**：可以聊天、倾诉，它会回应

### 设计检验标准（来自 Codex）

- **可晒**：用户拿得出手，最强的是好看的分享卡
- **可等**：有用户愿意等的东西（探险结果、周报、限定惊喜）
- **可收集**：纪念品、配饰、称号、限定卡片
- **可想念**：宠物表现出"它在想你"，而非系统通知

## 模块一：孵化与性格系统

### 1.1 品种（6 种）

孵化时随机分配。每种有不同的基础性格倾向和视觉风格。

品种示例（最终名称待定）：小企鹅、小恐龙、小狐狸、小兔子、小猫头鹰、小龙。

所有品种共用同一套场景模板和叠层合成逻辑，品种差异体现在：AI 基底立绘 + 语言风格 + 性格 baseline。如果某品种 AI 生图效果不稳定，可临时降级为预制图。

### 1.2 性格系统（双层架构）

**性格层**（稳定底色，长期慢变化）：

| 维度 | 低值 | 高值 | 影响 |
|------|------|------|------|
| 外向 ↔ 内向 | 安静、少话 | 主动说话、用词活泼 | 主动闲聊频率、用词风格 |
| 勇敢 ↔ 谨慎 | 怕生、犹豫 | 主动探索、不怕新事物 | 探险故事风格、面对未知的反应 |
| 嘴馋 ↔ 克制 | 对食物无感 | 总想吃、谈话常提食物 | 喂食反应、食物相关对话 |
| 好奇 ↔ 安定 | 喜欢待在原地 | 对一切感兴趣、问东问西 | 探索欲、提问频率 |
| 直球 ↔ 委婉 | 拐弯抹角、暗示 | 直接表达、不藏着 | 撒娇方式、表达情感的方式 |

**亲密度层**（反映当前关系，短期波动）：

- 独立数值 0.0-1.0
- 高频互动涨亲密度，忽视掉亲密度
- 影响：称呼进化、吃醋/黏人/嘴硬等行为、语音触发条件
- 亲密度断档时触发关系变化通知

### 1.3 初始值生成

```
final_trait = species_baseline + random_offset + hatching_offset
```

- `species_baseline`：每个品种的固定倾向值（如小狐狸偏委婉 0.7）
- `random_offset`：±0.15 随机偏移
- `hatching_offset`：孵化塑形阶段（前 3 次互动）的微调
- 最终值裁剪到 [0.1, 0.9]

### 1.4 孵化塑形

孵化不只是随机抽卡，用户前 3 次互动影响初始偏向：

- 更常"陪它说话" → 外向 +
- 更常"安抚它" → 直球方向微调
- 更常"鼓励它去看看外面" → 勇敢 +

用户感受："不是我抽到了一只宠物，是我把它孵成了现在这样。"

### 1.5 互动塑造（learned_offset）

```
displayed_trait = species_baseline + learned_offset
```

- 每次喂食：嘴馋 +0.005
- 每次探险：勇敢 +0.005
- 每次玩耍：外向 +0.005
- learned_offset 上限：±0.20~0.25
- 每日变化上限：±0.015
- 长期不强化时，缓慢回归 baseline（每天 -0.002）

### 1.6 性格感知

- **不暴露数值**，只在跨区间时自然语言通知
- 区间划分：[0, 0.3] 低 / [0.3, 0.7] 中 / [0.7, 1.0] 高
- 跨档通知示例："它最近明显更敢出门了"、"它现在越来越会主动贴贴你了"
- 性格影响通过 AI system prompt 注入："你是一只偏内向(0.3)、很嘴馋(0.85)的小狐狸..."
- 不同区间有不同口头禅、说话长度、emoji 习惯
- **情绪反差**：低直球值的宠物会偶尔吃醋、嘴硬、假装不在意

## 模块二：AI 生图与视觉系统

### 2.1 三层架构

```
角色基底层（AI 生成，永久缓存）
    ↓
状态动作层（AI 生成，可重算，按成长阶段解锁）
    ↓ Pillow 叠加
场景背景层（模板库，按时间/地点切换）
    ↓ Pillow 叠加
装饰配饰层（解锁后叠上去，不重生图）
    ↓
输出：聊天图 256px / 分享卡 1024px
```

### 2.2 孵化生图流程

1. 用户完成孵化塑形（3 次互动）→ 确定品种 + 初始性格
2. 调用生图 API 生成 **基础立绘**（1 张）—— 用户等待 ~10 秒，看到"你的宠物正在破壳..."
3. 同时生成 **3 张核心状态图**（idle / happy / sleeping）
4. 剩余 5 张状态图（eating / bathing / playing / exploring / sick）**后台异步补齐**
5. 异步期间用预制兜底图顶替

### 2.3 成长解锁机制

不一次性生成全部状态图，而是按成长阶段解锁：

| 阶段 | 解锁内容 |
|------|----------|
| 孵化完成 | 基底立绘 + idle + happy + sleeping |
| baby → child | 解锁 eating、bathing 专属图 |
| child → teen | 解锁 playing、exploring 专属图 |
| teen → adult | 解锁 sick 专属图 + 成年立绘 |

这同时解决了成本分散和养成激励两个问题。

### 2.4 提示词结构

```
cute {style}, chibi {species}, {color_palette},
{accessory}, {expression}, {action_pose},
white background, 256x256, consistent style
```

- 每次生成保存完整参数：model 版本、prompt、negative prompt、seed、参考图 ID、生成时间
- 状态图使用 image-to-image / reference image 模式，以基底立绘为参考，确保同一宠物跨状态视觉一致

### 2.5 生图失败处理

- 每个品种准备一套默认预制兜底图
- 生成时多出 2-3 张候选，自动筛选最佳
- 失败后静默降级到预制图，不阻断用户体验
- 后台定时重试失败的生图任务

### 2.6 场景化静帧

- 不只是角色 PNG，角色 + 场景背景（Pillow 模板合成）
- 早晨/午后/夜晚不同背景
- 探险地点各有独立场景
- 配饰解锁后叠层合成，不重生图

### 2.7 连环分镜（仅限仪式节点）

- 仅在 S 级高光时刻使用（探险归来、成就解锁、成年礼、纪念日）
- 发送 2-3 张连续静帧，配合 typing 节奏
- 日常互动不用分镜，避免刷屏

### 2.8 typing 节奏表演

- 利用 sendtyping API 制造期待感
- 关键互动节奏：先"正在输入..." → 文字消息 → 短暂停顿 → 图片
- 长时间 AI 处理时每 5 秒发一次 keepalive
- 日常简单回复不加刻意延迟

## 模块三：额度系统与商业模式

### 3.1 双能量系统

**陪伴能量**（免费层）：
- 每日自动恢复
- 覆盖：日常聊天、基础状态回复、简单主动问候、基础养成反馈
- 主动闲聊默认走预制模板，关键节点才调 AI
- 目标：陪伴不断，用户永远能和宠物说话

**创作能量（星星）**（付费层）：
- 充值获得
- 用于：生图、重绘、语音、精装分享卡、探险剧情长文、周报
- 前台明确显示消耗："10 星星 = 1 次重绘"、"3 星星 = 1 张成就卡"

### 3.2 初始额度分配（每人 ¥2）

| 用途 | 预算 |
|------|------|
| 孵化首发（基底 + 3 张状态图） | ¥0.5-0.7 |
| 成长解锁图（5 张，分阶段生成） | ¥0.7-1.0 |
| 文本对话 + 日记周报 | ¥0.2-0.3 |
| 异常缓冲 | ¥0.1-0.2 |

### 3.3 渐进式降级（非一刀切）

| 级别 | 触发条件 | 表现 |
|------|----------|------|
| 轻度降级 | 星星 < 30% | AI 长回复变短回复，主动闲聊减少，history 缩短 |
| 中度降级 | 星星 < 10% | 不再生成新图片、精装卡片，周报改模板 |
| 深度降级（犯困模式） | 星星 = 0 且陪伴能量不足 | 基础模板回复，但仍能看状态/日记/收藏 |

犯困模式文案风格：
- "今天有点困，先陪你聊聊，等补充点星星再给你画新图~"
- "我还能陪着你，只是今天画不动了"
- 状态衰减暂停（不惩罚用户）
- 分享卡仍可查看历史卡片

### 3.4 成本控制策略

- 主动闲聊：50% 预制模板 + 50% AI（仅对高活跃用户开启 richer chitchat）
- 日记/周报：用更短的 prompt 和更便宜的模型
- 对话历史压缩到 10 轮（省 prompt token）
- 生图按成长阶段分散，不一次性烧完

### 3.5 充值机制

- V1 阶段：用户转账，手动充值（几十人规模够用）
- 预留接口，后续可对接支付
- 充值档位：¥2 / ¥5 / ¥10
- 法律合规问题（是否需要公司资质等）待后续单独讨论

## 模块四：传播系统

### 4.1 传播优先级

1. **性格签卡**（第一传播武器）
2. **宠物名片**
3. **高光时刻卡**
4. **邀请码机制**
5. **周报卡**
6. **成长册 H5**

传播核心文案：**"看看你会孵出什么样的宠物"**（欲望），而非"找朋友要邀请码"（门槛）。

### 4.2 邀请码

- 4 位码，字母+数字（36^4 = 167 万种，内测足够）
- 初始 1 个码
- 补码路径（可控稀缺）：
  - 轻度：完成孵化、养满 3 天 → 第 2 个码
  - 中度：分享一次卡片、触发一次高光 → 新码
  - 重度：连续 7 天、探险 5 地点 → 额外码
- 邀请奖励：双方各获星星 + 好友纪念小卡/限定贴纸
- 无码用户看到："现在是邀请制领养哦，去找已经养宠物的朋友要一张邀请卡吧~"

### 4.3 扫码领养流程

1. 入口二维码 → 添加 bot
2. 发送邀请码 → 验证通过
3. 进入孵化塑形（3 次互动）
4. 生成宠物 + 性格签卡
5. 引导分享性格签卡

### 4.4 分享卡片

**性格签卡**（孵化完成时生成）：
- 类似 MBTI 结果卡
- "你孵出了一只【勇敢直球小狐狸】！" + 性格描述 + 立绘
- 1024px 精美排版
- 最强拉新素材——别人看到会想"我也想试试"

**宠物名片**（随时可查看）：
- 立绘 + 品种名 + 3 个性格标签 + AI 签名
- 例：🦊「七七」— 嘴馋·勇敢·直球 — "今天也想偷吃零食呢"

**成就卡**（S 级高光时触发）：
- 成就图标 + 达成条件 + 宠物表情 + 稀有度标识

**周报卡**（每周日生成，给主人自己看为主）：
- 本周名场面 + 数据摘要 + 立绘
- 不默认每周引导分享

### 4.5 高光时刻分级

| 级别 | 事件 | 处理 |
|------|------|------|
| S 级 | 第一次叫你名字、满月纪念、成年礼、SSR 纪念品、第一次语音 | 分镜 + 卡片 + 自然分享提示 |
| A 级 | 性格跨档、第一次探险回来、第一件收藏 | 轻提醒 + 记录，不强推分享 |
| B 级 | 普通成就、小阶段成长 | 只记录到日记/成长册 |

分享提示风格（自然，非弹窗）：
- "这个瞬间我帮你存成卡片啦~"
- "这张卡好像很适合发给朋友看看"

### 4.6 成长册 H5（轻版 V1）

独立网页，每只宠物唯一 URL。V1 只包含：

- 宠物立绘
- 3 个性格标签
- 在一起多少天
- 最近一个高光时刻
- 收藏品数量
- 1 段代表性日记
- 邀请入口："想养一只？找 TA 的主人要邀请码"

技术：Python 静态页生成 或 轻量 Flask。

## 模块五：声音层（稀缺惊喜）

### 5.1 设计原则

语音不是常态，而是**稀缺的情感高光**。大部分互动仍然是图文，只有极特殊时刻才触发语音，制造惊喜感。

### 5.2 触发条件（仅 S 级时刻）

- 第一次叫主人名字
- 满月纪念
- 成年礼
- 亲密度突破关键阈值

### 5.3 技术方案

- TTS API 生成语音（待选型：国内 TTS 如火山引擎、阿里云）
- 每个品种有独立音色参数
- 生成后缓存，同一事件不重复生成
- 通过 iLink voice_item 发送
- 消耗创作能量（星星）

## 模块六：技术壁垒总结

| 壁垒 | 说明 |
|------|------|
| AI 生图一致性管线 | 提示词工程 + reference image + 风格控制，不是调个 API 就能复制 |
| 多维性格 + 互动塑造 | baseline/offset 架构，性格随养育变化但保持品种底色 |
| 三层视觉合成 | 基底 + 场景 + 配饰叠层，配饰不重生图 |
| 双能量经济系统 | 陪伴免费 + 创作付费，渐进降级不断链 |
| typing 节奏编排 | 消息发送有表演节奏，不是一坨砸出来 |
| 传播资产生成 | 多种卡片模板 + AI 文案 + 高清合成 |

## 验证方案

### 孵化与性格验证
- 孵化 3 只不同品种的宠物，确认视觉和性格差异明显
- 连续互动 7 天，观察性格漂移是否自然
- 验证跨区间通知是否在正确时机触发

### 视觉系统验证
- 对比同品种不同个体的基底立绘，确认差异度
- 验证叠层合成效果（角色 + 场景 + 配饰）
- 生成分享卡片并在手机上查看 1024px 效果

### 额度系统验证
- 模拟一个用户从孵化到额度耗尽的完整生命周期
- 验证渐进降级各阶段的体验是否自然
- 确认 ¥2 预算能否覆盖完整孵化 + 基本养育周期

### 传播验证
- 生成性格签卡，发到群聊，观察是否有人问"怎么领养"
- 邀请码全流程测试：生成 → 分享 → 新用户使用 → 双方获奖励
- 成长册 H5 在手机浏览器打开，检查加载速度和显示效果
